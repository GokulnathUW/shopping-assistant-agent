# PLANNING.md

## What This Is
A local-first shopping assistant agent. User describes what they want. The agent researches market context, discovers products from trusted editorial sources, and finds the best price. All data stays local.

**Core philosophy:** The agent assists. The user decides.

---

## Agent Flow

```
1. User submits a product query
2. LLM generates investigation questions from multiple angles (Ollama)
3. Tavily fetches market context for those questions
4. LLM synthesizes context, asks user clarifying preference questions (Ollama)
5. User responds, query is enriched
6. LLM generates a list of trusted editorial sources for this product category (Ollama)
7. Validate each domain is reachable before using
8. Tavily fetches full content from those trusted sources (include_domains + include_raw_content)
9. LLM generates product suggestions with reasoning (Groq)
10. User picks a product
11. SerpAPI google_shopping fetches current prices and retailers
12. User makes the purchase
```

---

## Tools and What They Do

| Tool | Used For |
|---|---|
| Tavily Search | Market context (step 3), product content from trusted sources (step 8) |
| SerpAPI `google_shopping` | Price and retailer lookup (step 11) only |
| Ollama `llama3.2:3b` | Routing, question generation, parsing, trusted source list |
| Groq `llama-3.1-70b-versatile` | Final product suggestion synthesis only |
| SQLite | Caching all API responses |

**Tavily budget: 2 calls per user query** (1 for market context, 1 for product discovery)

---

## Key Decisions

- **SerpAPI google engine dropped** — organic results dominated by stores and brands, not editorial reviewers
- **Trusted sources generated by LLM per query**, not hardcoded — covers any product category dynamically. Validated for reachability before use.
- **Groq called once** — synthesis node only. Everything else uses local Ollama.
- **No RAG, no vector DB** — SQLite caches API responses. No static document corpus exists to retrieve over.
- **MCP deferred** — no natural fit in current flow. Will wrap tools as MCP servers in a polish phase if needed for portfolio purposes.
